The `@' commands that appear in place of the first five fields are extensions.

Value

Explanation

@reboot

Run once, at startup

@yearly

Run once a year, "0 0 1 1 *"

@annually

(same as @yearly)

@monthly

Run once a month, "0 0 1 * *"

@weekly

Run once a week, "0 0 * * 0"

@daily

Run once a day, "0 0 * * *"

@midnight

(same as @daily)

@hourly

Run once an hour, "0 * * * *"

What is crontab

The crontab file contains instructions for the cron daemon. Every user have a their own crontab. These commands will be executed as the user. Rows that start with # are comments.

Access the crontab file with "$ crontab -e"

Root user can edit another users crontab file by adding the username ex: "$ crontab -e [username]"
